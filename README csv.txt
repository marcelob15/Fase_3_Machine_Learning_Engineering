Desempenho em Memória (RAM)Embora o CSV seja um formato de texto, a performance em memória RAM durante a execução do modelo é idêntica à do Parquet, desde que a carga seja feita com tipagem estrita.FormatoEstratégia de CargaUso de RAM (5.2M linhas)CSV Padrãopd.read_csv(file)~1.45 GBCSV Otimizadopd.read_csv(file, dtype=optimized_dtypes)174.47 MBParquetpd.read_parquet(file)174.47 MB2. Por que CSV com Tipagem Estrita?A escolha pelo CSV em vez do Parquet neste pipeline justifica-se por:Interoperabilidade: O arquivo flights_ready.csv pode ser aberto e validado em qualquer editor de texto ou software de planilha (Excel/LibreOffice) sem necessidade de bibliotecas específicas (pyarrow ou fastparquet).Transparência no ETL: Permite verificar visualmente se as transformações de Label Encoding e Feature Engineering foram aplicadas corretamente antes do treinamento.Simplicidade de Deploy: Reduz a quantidade de dependências externas no ambiente de produção, facilitando a portabilidade do código entre diferentes sistemas operacionais.3. Mitigação de Perda de MetadadosO principal "defeito" do CSV é não salvar o tipo da coluna (ex: ele não sabe que uma coluna é int8). Resolvemos isso utilizando um Dicionário de Tipagem Otimizada em todos os scripts de leitura:Pythonoptimized_dtypes = {
    'MONTH': 'int8', 
    'DAY_OF_WEEK': 'int8',
    'AIRLINE': 'category',
    'HOUR': 'int8',
    'DISTANCE': 'float32',
    'TARGET': 'int8'
}
df = pd.read_csv('flights_ready.csv', dtype=optimized_dtypes)
Essa abordagem garante que o pipeline de Machine Learning mantenha a velocidade e a leveza, sem abrir mão da universalidade do formato CSV.
